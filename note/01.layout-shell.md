You are a senior frontend developer creating a note editor layout for a PKM (Personal Knowledge Management) app called Memry.

## TASK
Create the main layout shell component with three distinct zones.

## THREE ZONES EXPLAINED

### Zone 1: Main Content Area
- Takes majority of width (flexible, grows to fill space)
- Centered content with max-width of 720px
- Generous padding (48px horizontal on desktop, 24px on mobile)
- Light background (warm off-white: #fafaf9)
- This is where note title, tags, info, content, backlinks will live
- Scrollable independently

### Zone 2: Outline Edge
- Thin vertical strip (32px wide) on right edge of main content
- Contains small horizontal bars representing document headings
- Each bar is clickable (navigates to that heading)
- Has an info button (ℹ️) at bottom
- Semi-transparent background
- Shows on hover OR can be pinned visible
- When hovered: expands to show outline panel (240px)

### Zone 3: Right Sidebar
- Fixed width: 320px on desktop
- Contains two tabs: "AI Agent" and "Related"
- Has tab buttons at top
- Content area below tabs (scrollable)
- Slightly different background (white: #ffffff)
- Subtle left border (1px, #e7e5e4)
- Can be collapsed/hidden (toggle button)
- When collapsed: only a thin strip with expand button visible

## RESPONSIVE BEHAVIOR

### Desktop (>1200px)
- All three zones visible
- Main content: flexible
- Outline edge: 32px (expands to 240px on hover)
- Right sidebar: 320px

### Tablet (768px - 1200px)
- Right sidebar hidden by default (toggle to show)
- When shown: overlays as slide-in panel from right
- Main content takes full width minus outline edge

### Mobile (<768px)
- Right sidebar accessible via bottom sheet or modal
- Outline edge hidden (accessible via button)
- Main content takes full width

## VISUAL SPECIFICATIONS

### Colors
- Main content background: #fafaf9 (warm off-white)
- Right sidebar background: #ffffff (white)
- Border color: #e7e5e4 (warm gray)
- Outline edge background: transparent, #f5f5f4 on hover

### Shadows & Borders
- Right sidebar: 1px left border, subtle shadow when floating
- Outline expanded panel: subtle shadow (0 2px 8px rgba(0,0,0,0.08))

### Transitions
- Sidebar collapse/expand: 200ms ease-out
- Outline edge expand: 150ms ease-out
- All hover states: 150ms ease

isOutlinePinned (boolean) - Keep outline visible

EMPTY STATE

ACCESSIBILITY

Focus management when sidebar opens/closes

Create a clean, well-structured layout component that:

## Implementation Notes

### Key Techniques Used

1. **CSS Grid/Flexbox Hybrid**
   - Main layout uses CSS Grid for three-column structure
   - Inner components use Flexbox for alignment

2. **Conditional Rendering**
   - Outline panel only renders when expanded
   - Sidebar uses transform for slide animation (GPU accelerated)

3. **State Lifted Up**
   - Layout shell manages visibility states
   - Child components receive callbacks for toggling

4. **Slot Pattern**
   - `{children}` prop for main content injection
   - Tab content rendered based on active tab

### Why These Choices

| Choice | Reasoning |
|--------|-----------|
| 720px max-width | Optimal reading width (65-75 characters per line) |
| 320px sidebar | Standard sidebar width, fits content comfortably |
| 32px outline edge | Minimal footprint, unobtrusive |
| Warm grays | Matches app aesthetic, easy on eyes |
| 200ms transitions | Fast enough to feel responsive, slow enough to perceive |
