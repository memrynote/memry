Content Area (Editor Placeholder)
Purpose
Create the main content editing area — a clean, distraction-free space for writing. This is a placeholder/wrapper that will integrate with a rich text editor library.

The Prompt
You are a senior frontend developer creating a content area component for a PKM app called Memry.

## TASK
Create a clean, paper-like content area that serves as the main writing space. This component is a wrapper/placeholder that will eventually integrate with a rich text editor (like TipTap, Slate, or ProseMirror).
Use 7context mcp when you need to make implementation for packages. such as tiptap etc.

## HEADING EXTRACTION (For Outline)

The content area needs to emit headings for the outline panel:
```typescript
interface HeadingInfo {
  id: string;           // Unique ID for scrolling
  text: string;         // Heading text
  level: 1 | 2 | 3;     // H1, H2, H3
  position: number;     // Vertical position in px
}

// Emit on content change
onHeadingsChange: (headings: HeadingInfo[]) => void;
```

## PROPS INTERFACE
```typescript
interface ContentAreaProps {
  content: string | JSONContent;   // Content data (depends on editor)
  placeholder?: string;
  readOnly?: boolean;
  autoFocus?: boolean;
  onContentChange: (content: string | JSONContent) => void;
  onHeadingsChange: (headings: HeadingInfo[]) => void;
  onSelectionChange?: (selection: SelectionInfo) => void;
  onLinkClick?: (href: string) => void;
  onInternalLinkClick?: (noteId: string) => void;
}

interface SelectionInfo {
  from: number;
  to: number;
  text: string;
  isEmpty: boolean;
}
```

Key Techniques Used

Editor Library Integration

This component wraps the editor library


Slash Command Menu

Use TipTap's suggestion extension
Or build custom with position tracking
Filter items as user types


Floating Toolbar

Use floating-ui for positioning
Show on selection change
Hide when selection is empty


Heading Extraction

Parse content on change
Extract heading nodes with positions
Debounce for performance
